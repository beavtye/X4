# Naming Rules

- Project folders use lowercase kebab-case.
- Active projects live in `projects/<slug>/`.
- Templates live in `projects/_template/`.
- Archived projects live in `projects/_archive/<slug>/`.
- Python files use snake_case.
- Config templates use `.example` before the extension.
- Branches use `feature/`, `fix/`, `docs/`, or `ops/`, followed by the project slug and ticket.
- Avoid spaces, temporary suffixes, and ambiguous names.
- Deployable releases use semantic versions and include a SHA256 manifest.
