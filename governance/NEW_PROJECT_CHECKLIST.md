# New Project Checklist

1. Choose a lowercase kebab-case slug.
2. Add the project to `projects/REGISTRY.yaml`.
3. Copy `projects/_template/` to `projects/<slug>/`.
4. Complete `PROJECT_MANIFEST.yaml`.
5. Write the project README before adding implementation files.
6. Define source, scripts, tests, config, deploy, docs, manifests, and tools boundaries.
7. Create the matching Linear project or issue group.
8. Define the Gmail handoff subject prefix when an external agent is used.
9. Confirm repository visibility and access scope.
10. Run a repository safety check before the first implementation commit.

A project is not considered active until the registry and manifest are both present.
